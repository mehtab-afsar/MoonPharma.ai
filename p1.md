

# MoonPharma eBMR — Complete End-to-End Demo Plan

## What You're Asking For (Clarified)

You want a **single, complete demonstration flow** that:

1. Starts from onboarding a new company
2. Walks through the **ontology configuration** — where the admin defines entities, attributes, processes, and relationships through an admin interface
3. These configurations get saved as **that company's config** (SQL-backed)
4. Then executes **every eBMR process** end-to-end — from creating a product to approving a finished batch
5. All of this packaged so that on **any given day**, you can demo it to a pharma company and they understand the full platform

---

#### The SQL Schema That Powers This

```sql
-- ============================================================
-- ONTOLOGY CONFIGURATION SCHEMA
-- This is the meta-layer that lets each company define
-- their own entities, attributes, processes, and relationships
-- ============================================================

-- ============================================================
-- 1. ENTITY TYPES
-- What "things" does this company track?
-- Default: Material, Equipment, Product, Area, Personnel
-- Company can add: Instrument, Vehicle, Utility, etc.
-- ============================================================

CREATE TABLE "EntityType" (
    "id"            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"         UUID NOT NULL REFERENCES "Organization"("id"),
    "code"          VARCHAR(50) NOT NULL,          -- e.g. 'material', 'equipment'
    "label"         VARCHAR(100) NOT NULL,         -- e.g. 'Raw Material', 'Equipment'
    "description"   TEXT,
    "icon"          VARCHAR(50),                   -- UI icon identifier
    "isSystem"      BOOLEAN DEFAULT false,         -- protected defaults
    "isActive"      BOOLEAN DEFAULT true,
    "sortOrder"     INTEGER DEFAULT 0,
    "createdAt"     TIMESTAMP DEFAULT NOW(),
    "updatedAt"     TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "code")
);

-- ============================================================
-- 2. ENTITY ATTRIBUTES (Custom Fields per Entity Type)
-- Each entity type can have custom attributes beyond defaults
-- e.g. Material might have: grade, vendor, pharmacopeia
-- e.g. Equipment might have: capacity, RPM range, power
-- ============================================================

CREATE TABLE "EntityAttribute" (
    "id"                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"             UUID NOT NULL REFERENCES "Organization"("id"),
    "entityTypeId"      UUID NOT NULL REFERENCES "EntityType"("id") ON DELETE CASCADE,
    "code"              VARCHAR(50) NOT NULL,       -- e.g. 'grade', 'vendor'
    "label"             VARCHAR(100) NOT NULL,      -- e.g. 'Pharmacopeia Grade'
    "dataType"          VARCHAR(20) NOT NULL,       -- text, number, boolean, date, 
                                                    -- select, multiselect, file
    "isRequired"        BOOLEAN DEFAULT false,
    "isSearchable"      BOOLEAN DEFAULT false,      -- appears in filters
    "defaultValue"      TEXT,                       -- default when creating new record
    "validationRule"    JSONB,                      -- e.g. {"min": 0, "max": 100}
    "selectOptions"     JSONB,                      -- for select/multiselect types
                                                    -- e.g. ["IP", "BP", "USP"]
    "unit"              VARCHAR(20),                -- e.g. 'kg', 'ml', '°C'
    "isSystem"          BOOLEAN DEFAULT false,
    "isActive"          BOOLEAN DEFAULT true,
    "sortOrder"         INTEGER DEFAULT 0,
    "createdAt"         TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "entityTypeId", "code")
);

-- ============================================================
-- 3. LOOKUP CATEGORIES (Enhanced from existing)
-- Dropdown values for select-type attributes
-- Each company manages their own taxonomy
-- ============================================================

CREATE TABLE "LookupCategory" (
    "id"            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"         UUID NOT NULL REFERENCES "Organization"("id"),
    "categoryType"  VARCHAR(50) NOT NULL,          -- e.g. 'material_type', 'area_class'
    "value"         VARCHAR(50) NOT NULL,          -- e.g. 'active', 'Class_A'
    "label"         VARCHAR(100) NOT NULL,         -- e.g. 'Active Pharmaceutical Ingredient'
    "parentId"      UUID REFERENCES "LookupCategory"("id"),  -- hierarchical categories
    "metadata"      JSONB,                         -- extra info per category
    "isSystem"      BOOLEAN DEFAULT false,
    "isActive"      BOOLEAN DEFAULT true,
    "sortOrder"     INTEGER DEFAULT 0,
    "createdAt"     TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "categoryType", "value")
);

-- ============================================================
-- 4. PROCESS TEMPLATES
-- Define manufacturing process types
-- e.g. Granulation, Compression, Coating, Packaging
-- Each process type has a sequence of step templates
-- ============================================================

CREATE TABLE "ProcessTemplate" (
    "id"            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"         UUID NOT NULL REFERENCES "Organization"("id"),
    "code"          VARCHAR(50) NOT NULL,          -- e.g. 'wet_granulation'
    "name"          VARCHAR(200) NOT NULL,         -- e.g. 'Wet Granulation Process'
    "description"   TEXT,
    "category"      VARCHAR(50),                   -- e.g. 'manufacturing', 'packaging'
    "estimatedDuration" INTEGER,                   -- minutes
    "isSystem"      BOOLEAN DEFAULT false,
    "isActive"      BOOLEAN DEFAULT true,
    "sortOrder"     INTEGER DEFAULT 0,
    "createdAt"     TIMESTAMP DEFAULT NOW(),
    "updatedAt"     TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "code")
);

-- ============================================================
-- 5. STEP TEMPLATES (within a process)
-- Reusable step definitions that can be added to any MBR
-- e.g. "Sifting", "Dry Mixing", "Binder Addition"
-- ============================================================

CREATE TABLE "StepTemplate" (
    "id"                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"             UUID NOT NULL REFERENCES "Organization"("id"),
    "processTemplateId" UUID NOT NULL REFERENCES "ProcessTemplate"("id") ON DELETE CASCADE,
    "stepNumber"        INTEGER NOT NULL,
    "name"              VARCHAR(200) NOT NULL,      -- e.g. 'Sifting of API and Excipients'
    "instruction"       TEXT NOT NULL,               -- detailed SOP instruction
    "estimatedDuration" INTEGER,                     -- minutes
    "requiresLineClearance" BOOLEAN DEFAULT false,
    "requiresVerification"  BOOLEAN DEFAULT true,    -- two-person rule
    "isActive"          BOOLEAN DEFAULT true,
    "sortOrder"         INTEGER DEFAULT 0,
    "createdAt"         TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "processTemplateId", "stepNumber")
);

-- ============================================================
-- 6. PARAMETER TEMPLATES (within a step)
-- What must be recorded at each step
-- e.g. "Mixing Speed" (number, RPM, 100-500)
-- e.g. "Granule Appearance" (text, visual check)
-- ============================================================

CREATE TABLE "ParameterTemplate" (
    "id"            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"         UUID NOT NULL REFERENCES "Organization"("id"),
    "stepTemplateId" UUID NOT NULL REFERENCES "StepTemplate"("id") ON DELETE CASCADE,
    "code"          VARCHAR(50) NOT NULL,
    "label"         VARCHAR(200) NOT NULL,          -- e.g. 'Mixing Speed'
    "dataType"      VARCHAR(20) NOT NULL,           -- number, text, boolean, select
    "unit"          VARCHAR(20),                    -- e.g. 'RPM', '°C', 'min'
    "specLimitMin"  DECIMAL,                        -- lower spec limit
    "specLimitMax"  DECIMAL,                        -- upper spec limit
    "targetValue"   VARCHAR(100),                   -- ideal value or text description
    "isCritical"    BOOLEAN DEFAULT false,          -- triggers OOS if out of range
    "isRequired"    BOOLEAN DEFAULT true,
    "sortOrder"     INTEGER DEFAULT 0,
    "createdAt"     TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "stepTemplateId", "code")
);

-- ============================================================
-- 7. IPC (In-Process Control) TEMPLATES
-- Quality checks performed during a step
-- e.g. "LOD Check" (Loss on Drying), "Hardness Test"
-- ============================================================

CREATE TABLE "IPCTemplate" (
    "id"            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"         UUID NOT NULL REFERENCES "Organization"("id"),
    "stepTemplateId" UUID NOT NULL REFERENCES "StepTemplate"("id") ON DELETE CASCADE,
    "name"          VARCHAR(200) NOT NULL,          -- e.g. 'Loss on Drying (LOD)'
    "method"        TEXT,                           -- test method reference
    "specLimitMin"  DECIMAL,
    "specLimitMax"  DECIMAL,
    "unit"          VARCHAR(20),
    "frequency"     VARCHAR(100),                   -- e.g. 'Every 30 minutes'
    "isCritical"    BOOLEAN DEFAULT false,
    "sortOrder"     INTEGER DEFAULT 0,
    "createdAt"     TIMESTAMP DEFAULT NOW()
);

-- ============================================================
-- 8. ENTITY RELATIONSHIPS
-- Define how entities relate to each other
-- e.g. Material → Equipment (processed_by)
-- e.g. Step → Equipment (requires)
-- e.g. Product → Material (contains)
-- ============================================================

CREATE TABLE "EntityRelationship" (
    "id"                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"             UUID NOT NULL REFERENCES "Organization"("id"),
    "sourceEntityTypeId" UUID NOT NULL REFERENCES "EntityType"("id"),
    "targetEntityTypeId" UUID NOT NULL REFERENCES "EntityType"("id"),
    "relationshipType"  VARCHAR(50) NOT NULL,       -- 'requires', 'produces', 
                                                    -- 'processed_by', 'contains'
    "label"             VARCHAR(100) NOT NULL,      -- e.g. 'is processed by'
    "reverseLabel"      VARCHAR(100),               -- e.g. 'processes'
    "cardinality"       VARCHAR(20) DEFAULT 'many_to_many', 
                                                    -- one_to_one, one_to_many, many_to_many
    "isRequired"        BOOLEAN DEFAULT false,
    "isSystem"          BOOLEAN DEFAULT false,
    "isActive"          BOOLEAN DEFAULT true,
    "createdAt"         TIMESTAMP DEFAULT NOW(),
    
    UNIQUE("orgId", "sourceEntityTypeId", "targetEntityTypeId", "relationshipType")
);

-- ============================================================
-- 9. EQUIPMENT-STEP MAPPING (which equipment for which step)
-- ============================================================

CREATE TABLE "StepEquipmentRequirement" (
    "id"                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"             UUID NOT NULL REFERENCES "Organization"("id"),
    "stepTemplateId"    UUID NOT NULL REFERENCES "StepTemplate"("id") ON DELETE CASCADE,
    "equipmentTypeCode" VARCHAR(50) NOT NULL,       -- references LookupCategory
    "isRequired"        BOOLEAN DEFAULT true,
    "notes"             TEXT,
    "createdAt"         TIMESTAMP DEFAULT NOW()
);

-- ============================================================
-- 10. ORGANISATION CONFIGURATION (Enhanced)
-- ============================================================

CREATE TABLE "OrgConfiguration" (
    "id"                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "orgId"                 UUID UNIQUE NOT NULL REFERENCES "Organization"("id"),
    
    -- Numbering
    "batchPrefix"           VARCHAR(10) DEFAULT 'B',
    "batchNumberReset"      VARCHAR(20) DEFAULT 'yearly',
    "deviationPrefix"       VARCHAR(10) DEFAULT 'DEV',
    "changeControlPrefix"   VARCHAR(10) DEFAULT 'CC',
    "mbrPrefix"             VARCHAR(10) DEFAULT 'MBR',
    
    -- Workflow
    "qaReviewStages"        INTEGER DEFAULT 2,
    "requireLineClearance"  BOOLEAN DEFAULT true,
    "autoDeviationOnOos"    BOOLEAN DEFAULT true,
    "criticalDeviationHold" BOOLEAN DEFAULT true,
    
    -- E-Signature
    "eSignatureMethod"      VARCHAR(20) DEFAULT 'pin_only',
    "sessionTimeoutMinutes" INTEGER DEFAULT 30,
    "failedLoginLockout"    INTEGER DEFAULT 5,
    
    -- Yield
    "defaultYieldMin"       DECIMAL DEFAULT 95.0,
    "defaultYieldMax"       DECIMAL DEFAULT 105.0,
    "defaultMaterialTolerance" DECIMAL DEFAULT 2.0,
    
    -- Ontology Flags
    "allowCustomEntities"   BOOLEAN DEFAULT false,  -- enterprise only
    "allowCustomProcesses"  BOOLEAN DEFAULT true,
    "maxCustomAttributes"   INTEGER DEFAULT 20,     -- per entity type per plan
    
    "createdAt"             TIMESTAMP DEFAULT NOW(),
    "updatedAt"             TIMESTAMP DEFAULT NOW()
);

-- ============================================================
-- INDEXES FOR PERFORMANCE
-- ============================================================

CREATE INDEX idx_entity_type_org ON "EntityType"("orgId");
CREATE INDEX idx_entity_attr_org ON "EntityAttribute"("orgId", "entityTypeId");
CREATE INDEX idx_lookup_org_type ON "LookupCategory"("orgId", "categoryType");
CREATE INDEX idx_process_template_org ON "ProcessTemplate"("orgId");
CREATE INDEX idx_step_template_process ON "StepTemplate"("processTemplateId");
CREATE INDEX idx_param_template_step ON "ParameterTemplate"("stepTemplateId");
CREATE INDEX idx_entity_rel_org ON "EntityRelationship"("orgId");

-- ============================================================
-- SEED: DEFAULT ENTITY TYPES (created for every new org)
-- ============================================================

-- These would be inserted via application code on org creation:
-- 
-- EntityType: material, equipment, product, area, personnel
-- 
-- Default attributes for 'material':
--   code, name, type (select), grade (select), vendor, 
--   storageCondition, reorderLevel
--
-- Default attributes for 'equipment':
--   code, name, type (select), model, serialNumber, 
--   manufacturer, location, calibrationDueDate, qualificationStatus
--
-- Default attributes for 'product':
--   name, genericName, strength, dosageForm (select), 
--   therapeuticCategory, shelfLife, storageConditions
--
-- Default attributes for 'area':
--   name, classification (select), location, parentArea
--
-- Default attributes for 'personnel':
--   (mapped to User table — not a separate entity)
```
